## システム要件

### 1. プロジェクト概要

子供服の在庫、サイズ、使用状況を効率的に管理し、家族間での情報共有や買い替え時期の把握を支援するシステム。

### 2. 技術スタック

#### 2.1. フロントエンド / Webアプリケーション

- **Framework:** Next.js (App Router)
- **Language:** TypeScript
- **ORM:** Drizzle ORM (データベース操作および型定義)
- **Deployment:** Vercel

#### 2.2. バックエンド API (特定ロジック/データ処理用)

- **Language:** Python 3.10+
- **Framework:** FastAPI
- **API Documentation:** Swagger UI (自動生成)
- **Deployment:** Render.com

#### 2.3. インフラ / データベース

- **BaaS:** Supabase (Free Tier)
  - **Database:** PostgreSQL
  - **Authentication:** Supabase Auth
  - **Storage:** Supabase Storage (子供服の写真保存用)
- **Version Control:** GitHub

### 3. 機能要件

#### 3.1. 衣服管理機能

- 衣服情報の登録・編集・削除（ブランド、サイズ、カテゴリ、季節、購入日、状態など）
- 子供ごとの衣服振り分け
- 衣服の画像アップロード・プレビュー

#### 3.2. 検索・フィルタリング機能

- サイズ別、季節別、カテゴリ別での絞り込み
- 在庫状況（現役、保管中、処分予定など）による検索

#### 3.3. ユーザー認証・認可

- Supabase Auth を利用したログイン/ログアウト機能
- 家族間でのデータ共有機能（マルチテナントまたはグループ共有）

### 4. 非機能要件

#### 4.1. 運用・コスト

- Supabase および Vercel の無料枠（Free Tier）の範囲内で運用する。

#### 4.2. ユーザビリティ

- スマートフォンからの利用を主眼に置いたレスポンシブデザイン。
- Swagger UI を活用し、フロントエンド・バックエンド間のインターフェース整合性を維持する。

#### 4.3. 開発効率

- Drizzle ORM による TypeScript の型安全なデータベース操作。
